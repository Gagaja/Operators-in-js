JavaScript has the following types of operators.

This section describes the operators and contains information about operator precedence.


The code consists of:

  • key words
  • operator
  • variables
  • data

The operators show the interaction of data between each other or allow us to manipulate data in some way.


At one point the operator works with one, two or three different data, for example,
-1 in this way we display the negative number 1 and the operator minus in this case only affects one data.


In case 2 + 5 we have 2 values ​​that the operator connects, plus always works with 2,
even if we have for example,
2 + 3 + 4 always at one point plus only includes the first following value,
e.g. 2 + 3 gives 5 and then 5 is added to 4.

There is only one operator who works with 3 data at the same time: it's a ternary operator that will work
after we finish the loop.


For example in the expression 5 + 3 5 and 3 are values,
+ is an operator that explains how they work together in the code.


If we write a code that checks whether 2 + 3 is equal to 5 we will do this like this:
3 + 2 == 5 In this case, first 3 and 2 are added, and then their value is compared to 5.


Operators do not need to be executed in this order, there are rules for which this is done
and this is called "operator's priority".


For that you already heard in mathematics, you probably know that the expression value is below
7: 1 + 2 * 3 Multiplication has a higher priority than
compilation and it is performed first.


And here as in mathematics you can put brackets to change the priority
(1 + 2) * 3 - now the value of the expression will be 9.
